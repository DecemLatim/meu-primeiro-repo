#!/bin/bask
echo "Neste exercício, você vai usar o que aprendeu sobre systemd para escrever um serviço que iniciará toda vez que o sistema entrar em modo multi-user. Este será um serviço muito simples que grava uma mensagem de log em /var/log/messages.  usando uma ferramenta de log. Como administrador, haverá diversas situações que você deve escrever seus próprios scripts e serviços." 
echo
echo "Este exercício deve ser feito em uma distribuição CentOS 7+ ou Red Hat 7, como root ou então usando o comando sudo para executar o comando como root."
echo
echo "Apesar deste exercício ultrapassar o escopo da prova, ele vai deixa-lo muito bem preparado para o exame, pois cobre vários itens de forma bem prática."
echo
echo "Crie um arquivo .service chamado "mylogger.service" no diretório /etc/systemd/system. Dica: você pode copiar o sshd.service de /usr/lib/systemd/system se preferir."
Tenha certeza de que seu script irá rodar somente DEPOIS que o sshd.service for iniciado.
Na sessão [Service], você pode remover tudo, exceto ExecStart. 
Usando a opção ExecStart, tenha certeza de que toda vez que o script for executado, ele adiciona a linha : logger -f /var/log/messages "Hello world!". (Dica: informe o caminho completo do logger:  /usr/bin/logger)
Na sessão [Install], tenha certeza que a opção WantedBy inicia o serviço mesmo que ele tenha sido executado. Quando o sistema entrar no modo multi-user.target completamente, salve e saia.
Use o systemctl para habilitar o serviço.
Use  systemctl para iniciar o serviço.
Veja o status do serviço mylogger.
Use o comando tail para verificar se o serviço mandou uma mensagem para o arquivo  /var/log/messages.
Reinicie o sistema.
Entre no sistema e verifique se o serviço está ativo, e se adicionou outra mensagem no arquivo /var/log/messages.
Envie uma mensagem para todos dizendo: "Funcionou!"
